Import inventory data

Upload existing records to UPDATE or new features to INSERT. The next page shows a diff preview — nothing is changed in the database until you confirm.

Supported formats:

  • Zip of the GeoJSON round-trip pair — landslides.geojson + landslide_polygons.geojson, the same shape this site exports.
  • Zip with just landslide_polygons(_flat).geojson — polygons carrying landslide-level properties. Polygons sharing a unique_name get grouped; one landslide is synthesized per group.
  • Single GeoJSON (.geojson) — Polygon/MultiPolygon features → flat-polygon mode; Point/null features → landslides-only.
  • GeoPackage (.gpkg) — first layer read.
  • KML / KMZ (.kml, .kmz).
  • Shapefile — zip .shp + .dbf + .shx + .prj together. Shapefile DBF truncates field names to 10 chars, so prefer .gpkg or .geojson when authoring data with attributes like landslide_type / landslide_class that share a 10-char prefix.

Non-GeoJSON formats must be in EPSG:4326 (WGS84 lat/lon). Reproject in QGIS first if needed.

If editing in QGIS: toggle editing OFF on the layer (Layer menu → Toggle Editing, or click the pencil icon) before uploading. Otherwise recent edits live in a .gpkg-wal sidecar that doesn't ride along with the upload — only changes already committed to the main file are sent.
